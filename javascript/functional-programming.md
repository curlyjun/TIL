# Functional Programming

- 프로그래밍 패러다임 중 하나.

- 모든 것을 함수로 작성

- How 가 아닌 What을 생각해라.

- fure function

  - 사이드 이펙트가 없는 함수로 이루어져야함.
  - 외부에 영향을 끼치면 안됨. (함수 내부에서 전역 변수를 가져다 쓴다던지..)

- for, while 문이 아닌 map, reduce, filter 사용

- **immutable**(불변성 유지)

  - 함수에 input 으로 들어간 값이 변형되면 안됨.
